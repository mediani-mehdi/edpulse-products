# Railway Deployment Notes

Two services in a single Railway project, both pointing at this repo.

## Backend service
- Root Directory: `/backend`
- Build Command: `npm install && npm run build`
- Start Command: `npm run start:prod`
- Env vars:
  - `FRONTEND_URL` — frontend service public URL (set after frontend is created)
- Networking: Generate public domain.

## Frontend service
- Root Directory: `/frontend`
- Build Command: `npm install && npm run build`
- Start Command: `npm run start`
- Env vars:
  - `VITE_API_URL` — backend service public URL (no trailing slash)
- Networking: Generate public domain.

## Order of operations
1. Create backend service first, generate domain.
2. Create frontend service, set `VITE_API_URL` to backend domain, generate frontend domain.
3. Go back to backend, set `FRONTEND_URL` to frontend domain. Backend redeploys.
4. Verify `<backend>/health` returns `{"status":"ok"}`.
5. Verify frontend page loads products with no CORS errors.
